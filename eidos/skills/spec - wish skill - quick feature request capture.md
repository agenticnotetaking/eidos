# /eidos:wish

## Target

Not every feature idea is ready for a spec.
Wishes capture "I want this eventually" without forcing structure or commitment.
They sit between a passing thought and a specced feature — persistent enough to not get lost, lightweight enough to not slow you down.

## Behaviour

- Args: feature description
- Creates `memory/wish - <timestamp> - <claim>.md`
- Minimal structure — description and optional motivation/context
- Commits immediately
- When specced out, rename `wish -` to `specced -` (preserving timestamp and claim), link to resulting spec
